# Правила создания диалогов для симулятора

## КРИТИЧЕСКОЕ ТРЕБОВАНИЕ

Перед созданием любого нового диалога для симулятора ОБЯЗАТЕЛЬНО:

### 1. Полное согласование с пользователем

- **Сценарий**: Предложить и получить одобрение сценария (маршрут, груз, особенности)
- **Структура**: Согласовать количество шагов, путей (master/reject), ключевые точки принятия решений
- **Детали**: Подтвердить специфические требования (сертификаты, температура, страховка и т.д.)

### 2. Проверка существующей системы

Перед созданием диалога ОБЯЗАТЕЛЬНО прочитать и проанализировать:

```
pages/scenarios-data-v3.js
```

Проверить:
- Текущую структуру диалогов
- Формат outcome объектов
- Ограничения симулятора
- Последние изменения в логике

### 3. Обязательные компоненты диалога

Каждый диалог ДОЛЖЕН содержать:

#### Outcome объект (для всех путей):
```javascript
outcome: {
  type: "success" | "failure",
  quality: "excellent" | "good" | "poor",
  rate: number,
  ratePerMile: number,
  relationship: "strengthened" | "maintained" | "damaged",
  dialogueTime: number,
  questionsAsked: number,
  weeklyLoads: number,
  feedback: "строка с обратной связью"
}
```

#### Структура путей:
- **master**: Основной путь успешных переговоров
- **reject_*_final**: Финальные пути отказа (только с outcome)

### 4. Критические правила работы с reject путями

⚠️ **КРИТИЧНО**: Все reject пути ДОЛЖНЫ быть финальными!

- **Формат**: `reject_*_final` (например: `reject_weak_final`, `reject_attitude_final`)
- **Структура**: Только ОДИН элемент с `brokerResponse` и `outcome`
- **Запрещено**: Промежуточные шаги в reject путях (это создает тупики!)
- **Переключение**: Можно переключать на reject_*_final пути с ЛЮБОГО шага master пути
- **Варианты 4-6**: Всегда ведут в финальные reject пути, не в промежуточные шаги

#### Правильная структура reject пути:
```javascript
reject_weak_final: [
  {
    brokerResponse: "I appreciate your interest, but I need someone with stronger negotiation skills for this load.",
    outcome: {
      type: "failure",
      quality: "poor",
      rate: 0,
      // ... остальные поля
    }
  }
]
```

#### Неправильная структура (ЗАПРЕЩЕНО):
```javascript
reject_weak: [
  { brokerQuestion: "...", suggestions: [...] }, // ❌ Промежуточный шаг
  { brokerQuestion: "...", suggestions: [...] }, // ❌ Еще шаг
  { brokerResponse: "...", outcome: {...} }      // ❌ Создает тупик!
]
```

### 5. Процесс создания

1. ✅ Согласовать сценарий с пользователем
2. ✅ Прочитать `scenarios-data-v3.js`
3. ✅ Создать структуру диалога в отдельном файле `DIALOG-[NAME].js`
4. ✅ Показать пользователю для проверки
5. ✅ После одобрения - добавить в `scenarios-data-v3.js`
6. ✅ Протестировать логику всех путей
7. ✅ Зафиксировать в Git

### 6. Запрещено

❌ Создавать диалог без согласования сценария
❌ Добавлять диалог напрямую в scenarios-data-v3.js без проверки
❌ Использовать промежуточные шаги в reject путях
❌ Пропускать обязательные поля в outcome
❌ Игнорировать структуру существующих диалогов
❌ Создавать reject пути без суффикса "_final"

## Шаблон для согласования

При запросе на создание диалога спросить:

1. **Маршрут**: Откуда → Куда, расстояние?
2. **Груз**: Тип груза, особенности (температура, хрупкость)?
3. **Сложность**: Какие специальные требования (сертификаты, страховка)?
4. **Пути отказа**: Какие сценарии отказа предусмотреть?
5. **Цель обучения**: Какие навыки должен развить пользователь?

Только после получения ответов на все вопросы - приступать к созданию.
