# РЕЖИМ КОНСТРУКТОРА - Сборка ответа из блоков

## ОПИСАНИЕ

Новый режим обучения, где пользователь собирает ответ диспетчера из готовых блоков текста, нажимая на них.

## КАК РАБОТАЕТ

### 1. Структура данных

Каждый шаг диалога содержит `responseBlocks` вместо `suggestions`:

```javascript
{
    brokerQuestion: "What's your MC number?",
    responseBlocks: [
        { 
            id: 1, 
            text: "MC 123456", 
            points: 30, 
            required: true, 
            category: "credentials" 
        },
        { 
            id: 2, 
            text: "FastLoad Logistics", 
            points: 20, 
            required: true, 
            category: "company" 
        },
        { 
            id: 3, 
            text: "Dry van in Chicago", 
            points: 25, 
            required: true, 
            category: "location" 
        },
        { 
            id: 4, 
            text: "empty since yesterday", 
            points: 10, 
            required: false, 
            category: "details" 
        }
    ],
    minScore: 75
}
```

### 2. Интерфейс

**Модальное окно содержит:**

1. **Заголовок**: "🧩 Соберите ответ из блоков"
2. **Счетчик баллов**: Показывает текущий балл и минимум
3. **Превью ответа**: Показывает собранный текст
4. **Блоки**: Кнопки с текстом, которые можно выбирать
5. **Кнопка отправки**: "✅ Отправить ответ"

### 3. Блоки

Каждый блок - это кнопка с:
- **Текстом** (что добавится в ответ)
- **Баллами** (+30, +20, -15 и т.д.)
- **Категорией** (credentials, company, location, bad, etc.)
- **Обязательностью** (required: true/false)

**Цвета блоков:**
- 🔴 Красная граница: Обязательный блок
- 🟢 Зеленая граница: Высокие баллы (20+)
- 🟡 Желтая граница: Средние баллы (10-19)
- ⚪ Серая граница: Обычные блоки
- 🔴 Красный текст: Плохие блоки (отрицательные баллы)

### 4. Логика выбора

1. Пользователь кликает на блок → блок добавляется в ответ
2. Повторный клик → блок удаляется из ответа
3. Выбранные блоки становятся полупрозрачными
4. Баллы суммируются в реальном времени
5. Превью показывает собранный текст

### 5. Отправка ответа

При нажатии "Отправить":
1. Проверка обязательных блоков
2. Проверка минимального балла
3. Определение качества:
   - `excellent`: балл >= minScore + 20
   - `good`: балл >= minScore + 10
   - `normal`: балл >= minScore
   - `weak`: балл >= minScore - 10
   - `fail`: балл < minScore - 10
