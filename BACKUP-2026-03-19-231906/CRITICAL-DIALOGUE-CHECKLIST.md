# 🚨 КРИТИЧЕСКИЙ ЧЕКЛИСТ ДЛЯ СОЗДАНИЯ ДИАЛОГОВ
## ПРОВЕРЯЙТЕ ПЕРЕД ДОБАВЛЕНИЕМ КАЖДОГО ДИАЛОГА!

---

## ⚠️ ПРАВИЛО #0: ЛАКОНИЧНОСТЬ И ЕДИНООБРАЗИЕ ТЕКСТА!

### ✅ Проверка длины текста:

```
□ Шаг 1: Проверьте сообщения брокера
   - Каждое сообщение: 2-4 предложения
   - Примерно 100-200 символов
   - Не слишком короткие, не слишком длинные

□ Шаг 2: Проверьте ВСЕ подсказки (suggestions)
   - Откройте КАЖДЫЙ шаг диалога
   - Посмотрите на все варианты ответов
   - Все подсказки должны быть примерно ОДИНАКОВОЙ длины
   - Оптимально: 2-3 предложения, 150-250 символов
   - Разница между самой короткой и длинной не более 50 символов

□ Шаг 3: Проверьте аналитику
   - Максимум 1-2 предложения
   - Примерно 80-150 символов

□ Шаг 4: Проверьте советы (dispatcherPrompt)
   - Максимум 1 предложение
   - Примерно 50-100 символов
```

### 💡 ПОЧЕМУ ЭТО ВАЖНО:
- Визуальный баланс - пользователь видит одинаковые по размеру варианты
- Удобство выбора - не нужно читать одну подсказку 5 секунд, а другую 1 секунду
- Качество определяется СОДЕРЖАНИЕМ, а не длиной текста

---

## ⚠️ ПРАВИЛО #1: НЕТ ТУПИКОВЫМ СИТУАЦИЯМ!

### ✅ Проверка перед добавлением диалога:

```
□ Шаг 1: Посчитайте сколько путей (paths) в диалоге
   - Только master? → Переходите к Шагу 2
   - Несколько путей? → Переходите к Шагу 3

□ Шаг 2: Если только один путь (master)
   - Откройте ПЕРВЫЙ шаг диалога
   - Найдите массив suggestions
   - Добавьте path: "master" к КАЖДОМУ варианту
   - Пример:
     {
         text: "...",
         quality: "good",
         path: "master",  ← ОБЯЗАТЕЛЬНО!
         analytics: "..."
     }

□ Шаг 3: Если несколько путей
   - Проверьте что существуют ВСЕ пути: master, good, normal, weak
   - Если какого-то пути нет → создайте его ИЛИ добавьте path к suggestions

□ Шаг 4: Тестирование
   - Запустите диалог
   - Выберите КАЖДЫЙ вариант ответа в первом шаге
   - Убедитесь что диалог продолжается
   - Если застревает → вернитесь к Шагу 2
```

---

## 🔍 БЫСТРАЯ ПРОВЕРКА

### Вопрос 1: Сколько путей в вашем диалоге?
- **Один путь (только master)** → Добавьте `path: "master"` ко всем suggestions
- **Несколько путей** → Убедитесь что все пути существуют

### Вопрос 2: Есть ли `path` в suggestions первого шага?
- **Да** → Отлично! Переходите к следующей проверке
- **Нет** → СТОП! Добавьте path прямо сейчас!

### Вопрос 3: Все ли варианты ответов ведут к существующим путям?
- **Да** → Диалог готов!
- **Нет** → СТОП! Исправьте перед добавлением!

## 💡 ПРИМЕРЫ

### ❌ ПЛОХО (СОЗДАЕТ ТУПИК):
```javascript
paths: {
    master: [...]  // Только один путь
},
suggestions: [
    { text: "...", quality: "master" },  // Нет path!
    { text: "...", quality: "good" },    // ТУПИК!
    { text: "...", quality: "weak" }     // ТУПИК!
]
```

### ✅ ХОРОШО (НЕТ ТУПИКА):
```javascript
paths: {
    master: [...]
},
suggestions: [
    { text: "...", quality: "master", path: "master" },
    { text: "...", quality: "good", path: "master" },
    { text: "...", quality: "weak", path: "master" }
]
```

## 📋 ФИНАЛЬНЫЙ ЧЕКЛИСТ

Перед добавлением диалога в систему:

- [ ] Проверил длину всех сообщений брокера (2-4 предложения)
- [ ] Проверил что ВСЕ подсказки примерно одинаковой длины (150-250 символов)
- [ ] Проверил количество путей
- [ ] Добавил `path` ко всем suggestions (если один путь)
- [ ] ИЛИ создал все необходимые пути (если несколько)
- [ ] Протестировал ВСЕ варианты ответов
- [ ] Убедился что диалог НЕ заходит в тупик
- [ ] Диалог готов к добавлению!
