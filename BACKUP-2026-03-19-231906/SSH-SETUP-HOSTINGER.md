# Настройка SSH/SFTP для Hostinger

## Шаг 1: Включите SSH доступ в Hostinger

1. Войдите в панель: https://hpanel.hostinger.com/
2. Выберите ваш сайт (dispatch4you.com)
3. Перейдите в **Advanced → SSH Access**
4. Если SSH отключен, нажмите **Enable SSH Access**

## Шаг 2: Получите SSH данные

В разделе SSH Access вы увидите:

- **SSH Host**: обычно IP адрес (например: `94.23.44.78`) или домен
- **SSH Port**: обычно `22` или `65002` (Hostinger часто использует нестандартный порт)
- **SSH Username**: обычно совпадает с FTP username (например: `u724602277`)
- **SSH Password**: тот же, что и FTP пароль

## Шаг 3: Проверьте SSH подключение локально

Откройте PowerShell и выполните:

```powershell
# Для порта 22
ssh -p 22 u724602277@94.23.44.78

# Для порта 65002 (если 22 не работает)
ssh -p 65002 u724602277@94.23.44.78
```

Если подключение успешно, вы увидите приглашение командной строки сервера.

## Шаг 4: Обновите секреты GitHub (если нужно)

Если SSH порт отличается от 22, добавьте новый секрет:

1. Откройте: https://github.com/Alex305cebo/dispatch4you-site/settings/secrets/actions
2. Нажмите **"New repository secret"**
3. Добавьте:
   - Name: `SSH_PORT`
   - Secret: `65002` (или ваш порт)

## Шаг 5: Тестирование

Я создал специальный workflow `deploy-ssh.yml` который:
- Тестирует подключение на порту 22
- Если не работает, пробует порт 65002
- Показывает подробные логи

Запустите его:
1. Перейдите: https://github.com/Alex305cebo/dispatch4you-site/actions
2. Выберите "Deploy to Hostinger (SSH)"
3. Нажмите "Run workflow"

## Возможные проблемы

### SSH доступ отключен в Hostinger
**Решение:** Включите SSH в панели Hostinger (Advanced → SSH Access)

### Порт заблокирован
**Решение:** Hostinger может использовать нестандартный порт (65002 вместо 22)

### IP GitHub Actions заблокирован
**Решение:** 
1. В Hostinger перейдите в Security → IP Whitelist
2. Добавьте IP диапазоны GitHub Actions:
   - `4.175.114.51/32`
   - `20.102.39.0/24`
   - `20.119.28.0/23`
   - Полный список: https://api.github.com/meta (поле "actions")

### Неправильный путь к папке
**Решение:** Проверьте путь через SSH:
```bash
ssh -p 22 u724602277@94.23.44.78
pwd  # Покажет текущую директорию
ls   # Покажет содержимое
```

Обычно путь: `/home/u724602277/public_html/`

## Альтернатива: Использование SSH ключей (более безопасно)

### 1. Создайте SSH ключ локально:
```powershell
ssh-keygen -t ed25519 -C "github-actions"
```

### 2. Добавьте публичный ключ на Hostinger:
```bash
ssh -p 22 u724602277@94.23.44.78
mkdir -p ~/.ssh
echo "ваш_публичный_ключ" >> ~/.ssh/authorized_keys
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
```

### 3. Добавьте приватный ключ в GitHub Secrets:
- Name: `SSH_PRIVATE_KEY`
- Secret: содержимое приватного ключа

### 4. Обновите workflow для использования ключа

## Проверка статуса

После настройки проверьте:
1. SSH подключение работает локально
2. Workflow успешно выполняется в GitHub Actions
3. Файлы появляются на сервере в `/public_html/`

## Нужна помощь?

Если ничего не работает:
1. Свяжитесь с поддержкой Hostinger для уточнения SSH настроек
2. Попросите разблокировать IP адреса GitHub Actions
3. Используйте альтернативный метод деплоя (FTP или ручной)
