# ✅ Решение проблемы: /documentation не работает

## 🔴 Проблема
Вы пытались открыть `file:///C:/DispatcherTraining/documentation` и получили ошибку:
```
Не удалось получить доступ к файлу
```

## 💡 Причина
Clean URL `/documentation` работает только через веб-сервер с Apache и `.htaccess`. 
При открытии через `file://` протокол (двойной клик на HTML) это не работает.

## ✅ Решения

### 🎯 Решение 1: Быстрый доступ (РЕКОМЕНДУЕТСЯ)

**Двойной клик на один из файлов:**

1. `БЫСТРЫЙ-ДОСТУП.bat` - все страницы в одном месте
2. `ОТКРЫТЬ-ДОКУМЕНТАЦИЮ.bat` - прямо к документации
3. `ОТКРЫТЬ-ТЕСТ.bat` - полный список ссылок

**Или откройте HTML файлы напрямую:**

- `БЫСТРЫЙ-ДОСТУП.html`
- `ОТКРЫТЬ-ДОКУМЕНТАЦИЮ.html`
- `test-local-navigation.html`

### 📄 Решение 2: Прямые ссылки на документацию

Откройте один из этих файлов двойным кликом:

```
pages/documentation-new.html     ← Новая версия (рекомендуется)
pages/documentation.html         ← Основная версия
pages/documentation-old.html     ← Старая версия
```

### 🚀 Решение 3: Запуск веб-сервера

Если хотите использовать Clean URLs (`/documentation`):

1. Двойной клик → `ЗАПУСТИТЬ-СЕРВЕР.bat`
2. Откроется браузер на `http://localhost:8000`
3. Теперь можно открыть `http://localhost:8000/documentation`

**Или установите XAMPP:**
1. Скачайте: https://www.apachefriends.org/
2. Скопируйте проект в `C:\xampp\htdocs\dispatch4you\`
3. Откройте: `http://localhost/dispatch4you/documentation`
