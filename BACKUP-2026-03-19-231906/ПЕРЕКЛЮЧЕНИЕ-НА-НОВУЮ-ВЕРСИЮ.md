# 🔄 Переключение dispatch4you.com на новую версию через Git

## ⚠️ ВАЖНО: Сейчас на домене старая версия сайта

Эта инструкция поможет безопасно переключить домен на новую версию с возможностью отката.

---

## 📋 План действий (15 минут)

1. Создать бэкап старой версии
2. Настроить Git репозиторий на Hostinger
3. Переключить домен на новую версию
4. Проверить работу сайта

---

## ШАГ 1: Бэкап старой версии (ОБЯЗАТЕЛЬНО!)

### 1.1 Войдите в hPanel
- Откройте: https://hpanel.hostinger.com/
- Выберите домен: **dispatch4you.com**

### 1.2 Откройте File Manager
- Найдите раздел **Files** → **File Manager**
- Перейдите в папку: `/domains/dispatch4you.com/`

### 1.3 Создайте бэкап
```
Текущая структура:
/domains/dispatch4you.com/
  └── public_html/  ← Здесь сейчас старая версия
```

**Действия:**
1. Кликните правой кнопкой на папку `public_html`
2. Выберите **Rename**
3. Переименуйте в: `public_html_OLD_backup_2026-02-28`
4. Нажмите **Save**

✅ Теперь старая версия сохранена!

### 1.4 Создайте новую папку public_html
1. Нажмите **New Folder**
2. Имя: `public_html`
3. Нажмите **Create**

✅ Готово! Теперь у вас пустая папка для новой версии.

---

## ШАГ 2: Настройка Git на Hostinger

### 2.1 Откройте раздел Git
- В hPanel найдите раздел **Advanced** → **Git**
- Или используйте поиск: введите "Git"

### 2.2 Создайте Git репозиторий
Нажмите **Create Repository** и заполните:

```
Repository Name: dispatch4you
Branch to deploy: main
Repository Path: /domains/dispatch4you.com/public_html
```

⚠️ **ВАЖНО:** Путь должен быть ТОЧНО таким:
```
/domains/dispatch4you.com/public_html
```

### 2.3 Включите Auto Deploy
- ✓ Поставьте галочку **Auto Deploy**
- Это позволит автоматически обновлять сайт при каждом `git push`

### 2.4 Нажмите Create

### 2.5 Скопируйте Git Remote URL
После создания вы увидите **Git Remote URL**, примерно такой:
```
ssh://u123456789@123.456.789.0:65002/home/u123456789/repositories/dispatch4you
```

📋 **СКОПИРУЙТЕ ЕГО** - он понадобится в следующем шаге!

## ШАГ 3: Подключение локального проекта к Hostinger

### 3.1 Откройте PowerShell
- Перейдите в папку вашего проекта: `C:\DispatcherTraining`
- Кликните правой кнопкой → **Open in Terminal** (или **Git Bash**)

### 3.2 Добавьте remote
Выполните команду (замените URL на ваш из Шага 2.5):

```powershell
git remote add hostinger ssh://u123456789@123.456.789.0:65002/home/u123456789/repositories/dispatch4you
```

### 3.3 Проверьте подключение
```powershell
git remote -v
```

Должно показать:
```
hostinger  ssh://u123456789@...  (fetch)
hostinger  ssh://u123456789@...  (push)
```

## ШАГ 4: Отправка новой версии на сервер

### 4.1 Первый push
```powershell
git push -u hostinger main
```

⏳ Подождите 30-60 секунд...

При первом подключении может запросить:
- **Пароль от Hostinger** - введите пароль от вашего аккаунта
- **SSH fingerprint** - напишите `yes` и нажмите Enter

### 4.2 Дождитесь завершения
Вы увидите примерно такое:
```
Enumerating objects: 283, done.
Counting objects: 100% (283/283), done.
...
To ssh://...
 * [new branch]      main -> main
```

✅ **Готово!** Новая версия загружена на сервер!

## ❓ Возможные проблемы и решения

### Проблема 1: "Permission denied (publickey)"
**Решение:** Нужно добавить SSH ключ

1. Сгенерируйте SSH ключ:
```powershell
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
```
Нажимайте Enter на все вопросы (оставьте пустой пароль)

2. Скопируйте публичный ключ:
```powershell
cat ~/.ssh/id_rsa.pub
```

3. В hPanel:
   - Откройте **Advanced** → **SSH Access**
   - Нажмите **Manage SSH Keys**
   - Нажмите **Add New Key**
   - Вставьте скопированный ключ
   - Нажмите **Add**

4. Попробуйте снова:
```powershell
git push hostinger main
```

---

### Проблема 2: "Repository not found"
**Причина:** Неправильный Git Remote URL

**Решение:**
1. Проверьте URL в hPanel → Git → Repository Details
2. Удалите старый remote:
```powershell
git remote remove hostinger
```
3. Добавьте правильный:
```powershell
git remote add hostinger ПРАВИЛЬНЫЙ_URL
```

---

### Проблема 3: Сайт показывает 404 или пустую страницу
**Причина:** Неправильный Deploy Path

**Решение:**
1. В hPanel → Git → Repository Settings
2. Проверьте **Repository Path**: должен быть `/domains/dispatch4you.com/public_html`
3. Если неправильно - удалите репозиторий и создайте заново

---

### Проблема 4: Сайт не обновляется после push
**Причина:** Auto Deploy выключен

**Решение:**
1. В hPanel → Git → Repository Settings
2. Убедитесь что ✓ **Auto Deploy** включен
3. Или обновите вручную: нажмите **Pull Changes**

---

### Проблема 5: "fatal: refusing to merge unrelated histories"
**Причина:** На сервере уже есть другой Git репозиторий

**Решение:**
```powershell
git push hostinger main --force
```
⚠️ Используйте `--force` только при первом push!
