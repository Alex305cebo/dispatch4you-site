# Отчет об исправлении мобильной адаптации courses.html

## Дата: 12 марта 2026

## Проблема
Контент обрезался справа на узких мобильных экранах (Samsung Galaxy S20 Ultra, 412px ширина).

## Причины
1. `padding-left: 110px` на списках `<ul>` в секции Learning Path - слишком много для мобильных
2. Недостаточно агрессивные правила для очень узких экранов (320-420px)
3. Отсутствие `word-break` для длинных текстов

## Внесенные изменения

### 1. Улучшена адаптация секции Learning Path (@media max-width: 480px)
- Убран `padding-left` на списках (было 110px, стало 0)
- Добавлен `word-break: break-word` для текстов в списках
- Уменьшены размеры шрифтов (h3: 18px, p: 13px, li: 12px)
- Уменьшены иконки с номерами (50px вместо 80px)
- Уменьшен padding карточек (20px 12px)
- Добавлен `overflow: hidden` для предотвращения выхода контента

### 2. Улучшена адаптация CTA секции (@media max-width: 480px)
- Уменьшен padding (25px 12px вместо 30px 15px)
- Добавлен `word-break: break-word` для заголовков и текстов
- Уменьшены размеры шрифтов (h2: 22px, p: 14px)
- Уменьшены кнопки (padding: 14px 20px, font-size: 14px)
- Добавлен `overflow: hidden`

### 3. Добавлен новый breakpoint @media (max-width: 420px)
Специально для устройств типа Samsung Galaxy S20 Ultra (412px):
- Еще более компактные отступы (padding: 10px)
- Уменьшенные шрифты (h1: 30px, h2: 24px, h3: 17px)
- Минимальные размеры элементов
- Дополнительные правила для Learning Path и CTA секций

### 4. Обновлен breakpoint @media (max-width: 360px)
- Еще более минимальные размеры для самых маленьких экранов
- Padding: 8px
- Шрифты: h1: 28px, h2: 22px, h3: 16px

## Результат
✅ Контент больше не обрезается на узких экранах
✅ Все тексты читаемы и помещаются в viewport
✅ Нет горизонтального скролла
✅ Поддержка экранов от 320px до 1920px+

## Протестированные разрешения
- 412px (Samsung Galaxy S20 Ultra) ✅
- 375px (iPhone) ✅
- 360px (Android) ✅
- 320px (Extra small) ✅

- `courses.html` - добавлены улучшенные media queries для узких экранов
